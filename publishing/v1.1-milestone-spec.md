# v1.1 Milestone Spec — accounts + cross-platform progress + web ads

Status: SPEC ONLY. Activate AFTER mobile v1.0 ships. Do NOT redirect the factory
to this until v1.0 (ad-supported mobile + 29 SEK IAP) is live in the stores.
When activating: copy this into GOAL.md (operator action).

## Sprint target
Logged-in users (Google / Facebook / email) get their quiz progress synced across
iOS, Android, and web. Web build serves Google AdSense ads. Existing v1.0 users'
local progress migrates into their account on first login (no data loss).

## Architecture decisions (locked)
- Backend/auth: **Supabase** (aligns with CLAUDE.md "Supabase if needed").
  Supabase Auth providers: Google OAuth, Facebook OAuth, email/password.
- Datastore: Supabase Postgres table `user_progress`:
    user_id (uuid, fk auth.users) | question_id | best_state | attempts |
    updated_at | plus per-user settings/entitlements (adsDisabled mirrors IAP).
- Anonymous-first: app works with NO login (v1.0 behavior preserved).
  Login is optional; progress is local until the user signs in.
- Sync model: on login, MERGE — per question take max(local, cloud) progress;
  settings/entitlements last-write-wins by updated_at. Then cloud is source of
  truth; local cache writes through.
- Migration: first login pushes existing local progress up, then merge as above.
- Web ads: Google AdSense, web-only via a Platform.OS-guarded <WebAds/> component;
  mobile keeps AdMob untouched. AdSense added AFTER the web site is live + approved
  (AdSense only approves live, trafficked sites — cannot gate release on it).
- adsDisabled (Remove-Ads IAP) must also suppress web AdSense for that account.

## Acceptance (executable when activated)
- supabase auth: Google + Facebook + email sign-in all return a session
- user_progress round-trips: device A progress visible on device B after login
- merge: conflicting local vs cloud resolves to max per question (unit test)
- anonymous path still passes the full v1.0 monetization test suite
- web build renders AdSense slots only on web, none when adsDisabled
- existing-local-user migration test: no progress lost on first login

## Multi-language expansion (v1.1 — large, do NOT block v1.0)
Cover major immigrant languages spoken in Sweden, with NATURAL professional-
quality translations (not machine-literal): Arabic, Persian/Dari, Somali,
Tigrinya, Kurdish (Sorani/Kurmanji), Polish, Bosnian/Croatian/Serbian,
Spanish, Turkish, Ukrainian, Russian, Finnish, Thai, English (+ Swedish base).
- Proper i18n architecture (message catalogs per locale; RTL support for
  Arabic/Persian/Kurdish); every user-facing string + question/option/
  explanation/source line localized; in-app language switch.
- Translation QA per language: native-level naturalness review, not MT dumps;
  a stilted translation is a defect. Pipeline + per-locale review gate.
- Folds with v1.0 SOURCE-EXPANSION: question DB grows from multiple
  authoritative public sources (cited per question) and learns structure/
  coverage from other countries' citizenship exams (see competitive research).
Reason: requirement from operator 2026-05-17; large enough to be its own
milestone — ship v1.0 (SV+EN) first.
