## Objective restated as concrete success criteria

The user asked for non-mechanical localization across the civic-test project's
translated languages. Concrete deliverables are:

1. For each translated/target language, gather useful official or public-service
   language samples into organized project folders.
2. Learn and document the register, culture, humor boundaries, and word-choice
   conventions for each language.
3. Check words/phrases in the current non-Swedish language surfaces for
   mechanical translation and culturally weak jokes.
4. Improve the actual website/app language, not just write planning docs.
5. Keep non-ready locales fail-closed until UI strings, question content,
   accessibility/runtime checks, and native review are complete.
6. PR and merge each completed round to `main` with validation evidence.

## Question-localization v8 evidence added on this branch

This branch implements the v8 question-bank localization pipeline from
`/Users/billy/Desktop/almost-swedish-question-localization-research-v8.zip` for
the UHR-published question set:

- `data/questionLocalizations.ts` holds target-language maps for q001-q172.
- `QUESTION_LOCALIZATION_PILOT_IDS` is exactly q001 through q169.
- `scripts/check-question-i18n-v8.js` verifies every listed question has
  non-empty localized question, option, and explanation text for all 12 picker
  locales (`sv`, `en`, `zh-Hant`, `zh-Hans`, `ar`, `ckb`, `fa`, `pl`, `so`,
  `ti`, `tr`, `uk`).
- `tests/content-question-i18n-v8-pilot.test.js` pins the current coverage at
  `172 questions, 12 locales` and includes a negative test for missing localized
  option text.
- `scripts/export-site-question-bank.js` exports localized `q`, `opts`, and
  `why` maps into generated `site/questions.js`; exporter parity is checked with
  `node scripts/export-site-question-bank.js --check`.
- `docs/localization/readiness.json` records target question content as
  `pilot_q001_q169_machine_assisted` while keeping target locales unavailable.
- `QUESTION_LOCALIZATION_REVIEW_STATUS` records every q001-q172 target-locale question translation as `machine_assisted` with `pending_native_review`, and `scripts/check-question-i18n-v8.js` rejects missing or prematurely approved metadata.
- `scripts/check-question-i18n-v8.js` now enforces the v8 true/false label table (`Prawda/Fałsz`, `Sax/Khalad`, `ڕاستە/هەڵەیە`, etc.) and caught/fixed q074/q091/q094 ckb/so/ti label drift.
- `scripts/check-question-i18n-v8.js` also validates the source localization shape before export: q001-q172 only, exact option-id parity with source questions, no stale localized option IDs, and correct-option IDs still present in the source option set.
- `scripts/check-question-i18n-v8.js` protects high-risk Swedish civic/proper terms such as `Riksdag`, `Skatteverket`, `Migrationsverket`, `Allemansrätten`, `Treriksröset`, `Smygehuk`, and `Kebnekaise`; generated localized explanations preserve any missing canonical token in parentheses for reviewer visibility.
- `scripts/check-question-i18n-v8.js` now also checks numeric fact preservation across localized question, option, and explanation segments while normalizing Arabic-Indic and Persian digits plus common thousands grouping styles before comparing.
- `docs/localization/review-packets/question-i18n-v8/*.tsv` provides
  per-locale native-review packets for the 10 target locales, generated by
  `scripts/export-question-localization-review-packets.js` and checked by
  `tests/content-question-i18n-v8-review-packets.test.js`.
- `scripts/export-question-localization-review-packets.js` refuses to export or
  check question review packets unless target question readiness remains
  fail-closed as `pilot_q001_q169_machine_assisted` with missing native review
  and a blocked release gate.
- `scripts/export-ui-preview-review-packets.js` rejects unsupported pass,
  passport, citizenship, or approval outcome promises before preview-only UI
  review packets can be trusted.
- `scripts/export-ui-preview-review-packets.js` now also refuses to export or
  check preview-only UI review packets unless every preview locale remains
  unavailable in `lib/i18n/locales.ts` and fail-closed in
  `docs/localization/readiness.json`.
- `scripts/export-ui-preview-review-packets.js` rejects malformed UI placeholder
  syntax such as JavaScript template placeholders before generated packets can
  hide missing `{placeholder}` interpolation review.
- `scripts/export-ui-preview-review-packets.js` verifies every preview locale's
  `sourceStyleGuide` and `sourcePhrasebook` paths still point at existing
  repo-relative reviewer guidance files.
