# 14 — MVP Scope

## MVP goal

Launch a trustworthy version that users can actually study with.

Do not overbuild.

## MVP must-have features

| Feature | Include |
|---|---:|
| Android | Yes |
| iOS | Yes |
| Swedish/English toggle | Yes |
| Chapter list | Yes |
| Practice quiz | Yes |
| Single-choice questions | Yes |
| True/false questions | Yes |
| Answer explanations | Yes |
| UHR references | Yes |
| Mistake review | Yes |
| Mock exam | Yes |
| Progress tracking | Yes |
| Daily goal | Yes |
| Streak | Yes |
| XP | Yes |
| Swedish TTS audio | Yes |
| AdMob | Yes |
| Privacy policy | Yes |
| Disclaimer | Yes |

## MVP should-have features

| Feature | Include if easy |
|---|---:|
| Flashcards | Yes if time |
| Badges | Yes if time |
| Premium no-ads | Yes if time |
| Advanced stats | No |
| Human audio | No |
| Remote content updates | No |
| Login/accounts | No |

## MVP no-go features

Do not build:

- AI tutor,
- live chatbot,
- AI question generation inside app,
- social/community features,
- user-uploaded content,
- complex backend,
- multilingual support beyond English at first,
- official-looking certificate,
- guarantee/pass prediction.

## MVP content scope

Launch target:

- 500 reviewed UHR-based questions.
- 13 chapter records.
- 100–150 glossary terms.
- 10 mock exams generated from question bank.
- All questions with Swedish and English.
- All questions with explanations.
- All questions with UHR references.

Minimum beta target:

- 100 reviewed questions.
- 3–5 chapters.
- full quiz flow.
- full explanation/reference flow.

## MVP quality bar

A smaller high-quality question bank is better than a large weak one.

Do not publish question if:

- source is unclear,
- correct answer is debatable,
- English translation is confusing,
- Swedish wording is wrong,
- reference is missing,
- explanation overclaims.
